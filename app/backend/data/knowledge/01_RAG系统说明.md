# RAG 检索增强生成系统说明

RAG（Retrieval-Augmented Generation）是一种将知识库检索与大模型生成结合的技术路线。系统不会完全依赖模型参数中的记忆，而是先从本地知识库中检索与用户问题相关的文本片段，再将这些片段作为上下文交给大模型生成答案。

## 核心流程

1. 文档导入：用户上传课程资料、项目文档、讲义、制度文件等。
2. 文档切块：系统将长文档切分为多个 chunk。
3. 向量化或关键词索引：系统为 chunk 建立检索索引。
4. 问题检索：用户提问后，系统从知识库中召回相关 chunk。
5. 上下文增强：系统将召回内容拼接到 prompt 中。
6. 答案生成：模型基于检索内容进行回答。
7. 来源展示：系统展示引用文档、chunk 编号和相关分数。

## 优点

RAG 适合私有知识问答场景，可以减少幻觉，降低重新训练模型的成本，并且可以随着文档更新快速同步知识。

## 局限

RAG 对文档质量、切块策略、检索算法和 prompt 模板比较敏感。如果知识库中没有相关内容，模型仍可能回答不准确。因此系统需要提供“仅基于知识库回答”的开关。
