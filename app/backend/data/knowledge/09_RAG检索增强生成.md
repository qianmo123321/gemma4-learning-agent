# RAG 检索增强生成

## RAG 是什么
RAG 是 Retrieval-Augmented Generation 的缩写，意思是检索增强生成。系统先从知识库中检索相关内容，再把这些内容作为上下文交给大模型生成回答。

## 为什么需要 RAG
大模型参数中的知识可能过时、不完整或不适合私有资料。RAG 允许模型使用本地课程资料、项目文档、讲义和笔记，提高回答的可追溯性。

## RAG 基本流程
文档导入、文档切块、建立索引、用户提问、检索相关 chunk、拼接上下文、大模型生成答案、展示来源和证据。

## Chunk
Chunk 是文档切分后的文本片段。Chunk 太长会影响检索精度，太短会丢失上下文。常见做法是设置 chunk_size 和 overlap。

## RAG 不适合的问题
如果知识库没有相关内容，RAG 不能凭空保证正确。此时系统应提示知识库依据不足，而不是编造答案。
